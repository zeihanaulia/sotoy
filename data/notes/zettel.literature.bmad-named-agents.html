BMAD Named Agents sebagai literature note

Pertanyaan yang dibuka

  1. Mengapa BMAD memberi nama agent mereka?
  2. Bagaimana named agents membantu mengurangi prompt chaos?
  3. Apa peran customization dalam pengalaman AI coworker?

Ringkasan literatur

Halaman BMAD Named Agents menjelaskan bahwa nama seperti Mary, John, Winston, dan Amelia bukan sekadar persona. Mereka adalah cara menurunkan beban kognitif user, menjaga kontinuitas perilaku AI, dan membuat peran AI coworker menjadi lebih stabil.

BMAD membagi experience agent menjadi tiga primitive:

  • skill — kemampuan diskret seperti brainstorming, PRD, atau code review,
  • identity — harapan perilaku dan gaya interaksi,
  • customization — parameter yang bisa disesuaikan dengan tim.

Named agents membuat workflow lebih murah secara mental dan memberi struktur pada interaksi AI-native.

Ide penting

  • Named agents bukan gimmick; mereka adalah model interaksi.
  • Memberi nama dan peran membantu user tahu apa yang diharapkan saat memanggil agent.
  • Customization bukan sekadar personalisasi; ia menjadi bagian struktural dari sistem.

Interpretasi

Catatan ini menunjukkan bahwa BMAD melihat AI colleague sebagai pengalaman yang harus dikodifikasi. Dengan named agents, BMAD menempatkan AI bukan sebagai feature, tapi sebagai komponen operasional dalam workflow engineering.
