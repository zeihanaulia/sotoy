> Agent juga butuh mental model repo. Onboarding docs membantu agent memahami konteks bisnis, tujuan produk, aturan arsitektur, dan norma tim.

Catatan gue:

- Onboarding docs untuk manusia dan agent berbeda fungsi, tapi keduanya memerlukan konteks yang sama.
- Tanpa onboarding yang jelas, agent mungkin tahu struktur file tetapi tidak tahu mengapa sistem dibangun seperti ini.
- Onboarding sebaiknya mencakup why, what, how, dan rules agar agent punya landasan keputusan.

Implikasi:

- Onboarding docs bukan sekadar halaman setup; ia adalah artefak konteks untuk executable governance.
- Agent yang membaca onboarding lebih mungkin menghasilkan perubahan yang konsisten dengan tujuan bisnis dan arsitektur tim.
- Perhatian harus diberi pada format dan urutan onboarding agar mudah dipakai oleh agent.
